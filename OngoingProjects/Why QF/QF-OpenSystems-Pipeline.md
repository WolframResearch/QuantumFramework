| D3 | Axis of the series: **question type** (characterize, locate a transition, exploit structure, close the algebra, couple to classical control, change representation), laddered recognizable $\to$ consequential. | Per the showcase-design lessons: pick the axis from the claim's mechanism. ## 2. The QuTiP baseline (what "unique" is measured against)

QuTiP 5.x has: `mesolve`/`mcsolve`/`brmesolve`/`smesolve` (all numeric), numeric `liouvillian`/`propagator` superoperators, `steadystate`, time-dependence via `QobjEvo` (including interpolated array coefficients), `to_kraus`/`to_choi` on numeric superoperators, continuous-variable Wigner functions for plotting. It has **no** symbolic layer at all: no closed-form $\rho(t)$, no symbolic rates, no exact transition times, no event-triggered dynamics inside the solver, no discrete-Wigner master-equation representation, no matrix-valued-rate (Kossakowski) input. Examples were designed to land in those gaps and *only* claim those gaps.

## 3. Assumptions log

| # | Ex | Assumption | Held? | Correction |
|---|---|---|---|---|
| B1 | 1 | QF's named `"J-"` is the textbook $\sigma_-$ that empties the excited state $\lvert 1\rangle$. | **No** | Probed: under `"J-"` the population of $\lvert 0\rangle$ decays ($\lvert 0\rangle$ = spin-up in the J convention). The explicit matrix `{{0,1},{0,0}}` ($\sigma_- = \lvert 0\rangle\langle 1\rvert$) gives the textbook direction ($\rho_{22}$ decays). The showcase defines $\sigma_-$ explicitly and never uses `"J±"` for qubit decay. |
| B2 | 3 | The Kossakowski matrix-rate form `"Liouvillian"[H, Ls, β]` works with a symbolic $\beta$. | **No (kernel bug)** | `NamedOperators.m:908` guards each cross term with `If[#1 == 0, 0, ...]`; for symbolic rates the `If` is undecidable and stays unevaluated, producing a `Thread::tdlen` mess. Numeric $\beta$ works (doc-page path). Separately, `QuantumEvolve[H, Ls -> β, ...]` with a *matrix* $\beta$ silently takes elementwise `Sqrt` of the matrix (`QuantumEvolution.m:77`), wrong, not guarded. Filed as a spawn-task. Symbolic route: diagonalize $\beta$ by hand into collective modes $L_\pm = (L_1 \pm L_2)/\sqrt2$ with rates $\gamma \pm \gamma_{12}$, which is *better* pedagogy anyway (the eigenmodes are the super/subradiant channels). |
| B3 | 6 | Depolarizing noise gives a finite Wigner-negativity death time. | **No (physics)** | Depolarizing contracts the Bloch vector toward the maximally mixed state; $W_{\min}(t) = -e^{-\gamma t}/(4\sqrt3)$ stays negative for all finite $t$. Amplitude damping (flow toward $\lvert 0\rangle$, whose Wigner function is non-negative) gives a genuine finite classicalization time. Switched the noise model. |
| B4 | 6 | `QuantumEvolve` solves the phase-space (real-vector) ODE symbolically like it does the complex ones. | **No** | With symbolic $\gamma$ the `DSolveValue[..., s[t] ∈ Vectors[4, Reals], t]` call returns unsolved and `QuantumEvolve::error` fires (the same diagonal system with numeric rate must be re-probed; see B5). Also found: `QuantumEvolution.m:173` filters the symbolic-path options with `FilterRules[{opts}, Options[NDSolveValue]]`, so DSolve-only options (e.g. `Assumptions`) are **silently dropped**, `$Assumptions` is the only way in. Workaround in the showcase: show the phase-space ODEs via `"ReturnEquations" -> True` (verified: $\dot W = \mathrm{diag}(0,-\gamma,-\gamma,-\gamma)W$ for depolarizing, the master equation as a classical rate equation), and get the closed form by transforming the symbolically-evolved $\rho(t)$ into the Wigner basis. |
| B5 | 6 | gamma=1 rational rate lets the Reals-domain DSolve succeed. | **No** | Even with the rate set to exact 1, `DSolveValue[..., s[t] ∈ Vectors[4, Reals], t]` returns unsolved (v9). The `Vectors[n, Reals]` domain itself appears to be the blocker, independent of symbolic rates. Showcase route: `"ReturnEquations"` for the native phase-space form, closed form via the Wigner transform of the symbolically evolved $\rho(t)$. |
| B6 | 2 | `Solve[C(t) == 0, t]` returns the death time cleanly. | Partly | It returns the right `ConditionalExpression` ($t^* = \ln\frac{1}{1-\cot\theta}/\gamma$ valid iff $4\theta > \pi$) but leaks `Solve::ztest1`. Message-free form: `Reduce` on the factored coherence-vs-population equation under $\pi/4 < \theta < \pi/2$. |
| B7 | 1-4 | Symbolic 2-qubit Lindblad solves are fast enough to ship. | **No for Ex3** | The 1-qubit and ESD solves return in minutes; but the collective-mode dissipator with even **one** symbolic rate parameter in the generator ($r$, rates $1\pm r$) did not return in ~13 min (three attempts: full $\gamma,\gamma_{12},\omega$; lean $\gamma,\gamma_{12}$; units-of-$1/\gamma$ single symbol $r$). The structural difference vs ESD: ESD's generator carries one symbol $\gamma$ and is triangular in matrix units; the collective dissipator couples the single-excitation coherences with two distinct rates. **Resolution:** symbolic physics from `Eigensystem[β]` (rates $1\pm r$, exact, instant) + exact-rational runs ($r=9/10$: singlet $e^{-t/10}$, triplet $e^{-19t/10}$) + the exact singular point $r=1$ (singlet drift = exact zero matrix). Kossakowski numeric path cross-checks to $7.4082\times10^{-1}$ on both sides. All verified in v8. |
| B8 | 5 | The `WhenEvent` example runs like any numeric `QuantumEvolve`. | **No (bisected to a hard kernel crash)** | The "license error" message was wolframscript's report of the kernel dying *inside* `NDSolveValue`. Bisection (t2-t7): (a) `WhenEvent` watching `Indexed[s[t], {2,2}]` with a `Sow` action: **works, message-free**; (b) time-triggered `WhenEvent` flipping a `DiscreteVariables` drive: solves, but the event-discontinuous `InterpolatingFunction` then trips `ExpandInterpolatingFunction` (`Interpolation::inddp`, duplicated grid point at the switch); (c) drive as a continuous auxiliary variable with a jump assignment: `NDSolveValue::ndsz` stiffness stop at the event; (d) **state-watching event + `DiscreteVariables` flip: kernel hard-crashes** (with or without `"ReturnSolution"`), an NDSolve-level crash, not QF post-processing. Final design: calibrate (Sow the crossing time, state-watching event) then control (time-triggered discrete switch at that time) with `"ReturnSolution" -> True` to skip the `inddp`-leaking re-packaging; verified message-free, $P_1$ rises to the threshold then decays. |
| B9 | 3 | The Heisenberg-picture form deserves its own Ex3 cell. | Dropped | `QuantumEvolve[H, lindblad, observable, t]` computes the adjoint evolution via the **doubled** propagator (`QuantumEvolution.m:16-21`); for 2 qubits that is a 16×16 symbolic `DSolve` (the thing B7 showed doesn't return). The single-qubit Heisenberg check remains a candidate for Ex4; not load-bearing for Ex3. |
| B10 | 6 | "Negative entry of the QF Wigner vector" = magic negativity, for a qubit. | **No, twice** | (a) The even-$d$ `"Wigner"[2]` 4-vector components are Fano/Bloch-type coordinates (one is $\propto -r_z$), not phase-point quasi-probabilities; its "negativity" never dies and means nothing resource-theoretic. (b) `"WignerMIC"[2]` *is* a genuine quasi-probability (sums to 1) but it is a MIC-POVM probability, **non-negative for every state** (verified: the $T$-state gives $\{(6\mp\sqrt3)/24\}$, all positive), so no negativity story there either; and the qubit Wootters function of the $T$-state pointing at a tetrahedron corner is positive anyway (qubit discrete Wigner is not a magic monotone). **Resolution: odd $d$.** Qutrit strange state $(\lvert1\rangle - \lvert2\rangle)/\sqrt2$ in `"Wigner"[3]`: $W(0) = \{-\tfrac13, \tfrac16\times8\}$, sum 1, the maximal qutrit negativity, exactly the Howard-2014 setting. Depolarizing via the 8 Heisenberg-Weyl jumps at $\gamma/9$ relaxes $\rho$ to $I/3$ at rate $\gamma$; $W_0(t) = \tfrac19 - \tfrac49 e^{-\gamma t}$; `Reduce` gives the negativity death time **exactly** $t^* = 2\ln2/\gamma$. Bonus display: in the phase-space picture the depolarizing generator is a doubly-stochastic Markov matrix (columns sum to 0). |

## 4. Verified example map (running)

| Ex | Question type | Unique mechanism | Verified result |
|---|---|---|---|
| 1 | Characterize | One symbolic solve covers **all** initial states and **all** rates at once; the physicality bound falls out | $\rho(t)$ for symbolic $\rho_{ij}$, $\gamma_1$, $\gamma_\phi$, $\Omega$; populations $e^{-\gamma_1 t}$, coherence $e^{-(\gamma_1/2 + 2\gamma_\phi + i\Omega)t}$; `Reduce` $\Rightarrow T_2 \le 2T_1$ with equality iff $\gamma_\phi = 0$ |
| 2 | Locate a transition | Exact entanglement-sudden-death time, with its own validity condition emerging from the solver | $C(t) = 2\sin\theta\,e^{-2\gamma t}(e^{\gamma t}(\cos\theta-\sin\theta)+\sin\theta)$; $t^* = \frac{1}{\gamma}\ln\frac{1}{1-\cot\theta}$ iff $\theta > \pi/4$; QF's numeric `Concurrence` matches to $10^{-15}$ |
| 3 | Exploit structure | Symbolic collective rates $1 \pm r$ from `Eigensystem[β]`; exact-rational runs; the DFS exactly at the singular point $r=1$; Kossakowski matrix form (numeric) | Singlet $e^{-t/10}$, triplet $e^{-19t/10}$ at $r = 9/10$ exact; singlet drift at $r=1$ = exact zero matrix; numeric $\beta$-matrix path agrees ($0.740818$ both sides) |
| 4 | Close the algebra | The propagator, the named channel, and the circuit are the same object algebra | $S(t)$ closed form; semigroup `True` symbolically; master equation $\equiv$ `QuantumChannel["AmplitudeDamping"[1-e^{-\gamma t}]]` `True`; Ramsey $P_0 = \tfrac12(1 + e^{-(\gamma+4\gamma_\phi)t/2}\cos\delta t)$ through gate→Lindblad→gate |

| 6 | Change representation | The Lindblad equation as a real Markov flow on discrete-Wigner quasi-probabilities (qutrit); exact negativity death time | $W(0) = \{-\tfrac13, \tfrac16\times8\}$; generator columns sum to 0; $W_0(t) = \tfrac19 - \tfrac49 e^{-\gamma t}$; `Reduce` $\Rightarrow t^* = 2\ln2/\gamma$ exactly |

## 5. Kernel findings worth upstreaming

1. **`"Liouvillian"` symbolic matrix rates** (`NamedOperators.m:908`): `If[#1 == 0, 0, ...]` undecidable for symbolic entries → unevaluated soup. Fix: `If[TrueQ[#1 == 0], ...]`.
2. **`QuantumEvolve[H, Ls -> β]` with matrix β** (`QuantumEvolution.m:25,77`): the rule form accepts a matrix as `gammas_List` and then computes `Sqrt[gammas]` elementwise, silently wrong dimensions. Should reject or route to the Kossakowski path.
3. **DSolve-only options dropped** (`QuantumEvolution.m:169-174`): the symbolic branch filters `opts` against `Options[NDSolveValue]`, so `Assumptions` etc. never reach `DSolveValue`.
4. **Phase-space symbolic solve fails** (B4/B5): `Vectors[n, Reals]` + symbolic rate appears to defeat `DSolveValue` where the equivalent complex-domain solve succeeds.
