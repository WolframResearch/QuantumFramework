# Root | [SpanFromLeft]

> [Root](https://reference.wolfram.com/language/ref/Root.html)[{*f*,*c*}] ‚Äî represents the exact root of the general equation `*f*[*x*]==0 near `*x*=*c*`.
> [Root](https://reference.wolfram.com/language/ref/Root.html)[{{*f*_1,‚Ä¶,*f*_*n*},{*c*_1,‚Ä¶,*c*_*n*}},*j*] ‚Äî represents the `*j*`$^{th}$ coordinate of the exact root of the system of equations `{*f*_1[*x*_1,‚Ä¶,*x*_*n*]==0,‚Ä¶,*f*_*n*[*x*_1,‚Ä¶,*x*_*n*]==0}` near `{*x*_1,‚Ä¶,*x*_*n*}={*c*_1,‚Ä¶,*c*_*n*}`.
> [Root](https://reference.wolfram.com/language/ref/Root.html)[*f*,*k*] ‚Äî represents the exact `*k*`$^{th}$ root of the polynomial equation `*f*[*x*]==0
> [Root](https://reference.wolfram.com/language/ref/Root.html)[{*f*_1,*f*_2,‚Ä¶},{*k*_1,*k*_2,‚Ä¶}] ‚Äî represents the last coordinate of the exact vector `{*a*_1,*a*_2,‚Ä¶}` such that `*a*_*i*` is the `*k*_*i*`$^{th}$ root of the polynomial equation `*f*_*i*[*a*_1,‚Ä¶,*a*_*i-1*,*x*]==0

## Details and Options

[Root](https://reference.wolfram.com/language/ref/Root.html) is also known as an algebraic number when *f* is polynomial with integer coefficients or a transcendental number when there is no such polynomial *f* possible.

[Root](https://reference.wolfram.com/language/ref/Root.html) is typically used to represent an exact number and is automatically generated by a variety of algebra, calculus, optimization and geometry functions.

[Root](https://reference.wolfram.com/language/ref/Root.html) represents an exact number as a solution to an equation `*f*[*x*]==0 with additional information specifying which of the roots is intended.

[Root](https://reference.wolfram.com/language/ref/Root.html) numbers can be used like any other numbers, both in exact and approximate computations.

[Root](https://reference.wolfram.com/language/ref/Root.html) numbers are formatted as `Root` where `*approx*` is a numerical approximation. An approximation to precision `*p*` can be computed using [N](https://reference.wolfram.com/language/ref/N.html)[*Root*,*p*].

For most uses, [Root](https://reference.wolfram.com/language/ref/Root.html) objects are automatically generated and can be directly used. For advanced uses, when your code is going to directly generate [Root](https://reference.wolfram.com/language/ref/Root.html) objects, a deeper understanding of the different representations is necessary.

There are two distinct mechanisms used to specify which root of an equation is represented, the neighborhood representation [Root](https://reference.wolfram.com/language/ref/Root.html)[{*f*,*c*}] and the indexing representation [Root](https://reference.wolfram.com/language/ref/Root.html)[*f*,*k*].

The root neighborhood representation [Root](https://reference.wolfram.com/language/ref/Root.html)[{*f*,*c*}] specifies the equation `*f*[*x*]==0 as well as the neighborhood rectangle centered at `*c*` and width $2 10^{-Accuracy[Re(c)]}$ and height $2 10^{-Accuracy[Im(c)]}$.

$$
\text{[Graphics]}
$$

The root neighborhood representation for systems [Root](https://reference.wolfram.com/language/ref/Root.html)[{{*f*_1,‚Ä¶,*f*_*n*},{*c*_1,‚Ä¶,*c*_*n*}},*j*] similarly specifies a system of equations `{*f*_1[*x*_1,‚Ä¶,*x*_*n*]==0,‚Ä¶,*f*_*n*[*x*_1,‚Ä¶,*x*_*n*]==0}` and a neighborhood given by the product of rectangles from `*c*_*i*` for the different coordinates.

The root neighborhood representation [Root](https://reference.wolfram.com/language/ref/Root.html)[{*f*,*c*,*m*}] specifies that `*f*[*x*]==0 has a root of multiplicity `*m*` in the neighborhood given by `*c*`. However, it may be a cluster of closely spaced roots and by refining the neighborhood `*c*`, i.e. higher precision root approximation, they may separate.

The root indexing representation [Root](https://reference.wolfram.com/language/ref/Root.html)[*f*,*k*] applies to polynomial functions `*f*` only. The indexing of roots takes the real roots first, in increasing order. For polynomials with rational coefficients, the complex conjugate pairs of roots have consecutive indices.

\text{[Graphics]}
$$

The root indexing representation for systems [Root](https://reference.wolfram.com/language/ref/Root.html)[{*f*_1,*f*_2,‚Ä¶,*f*_*k*},{*k*_1,*k*_2,‚Ä¶,*k*_*n*}] applies to triangular systems of polynomial equations only. Given equations `*f*_1[*x*_1]==0, `*f*_2[*x*_1,*x*_2]==0, `‚Ä¶`, `*f*_*n*[*x*_1,*x*_2,‚Ä¶,*x*_*n*]==0, we recursively define `*r*_1 as the `*k*_1$^{th}$ root of ``*f*_1[*x*_1]==0`, `*r*_2 as the `*k*_2$^{th}$ root of `*f*_2[*r*_1,*x*_2]==0 and finally `*r*_*n*` as the `*k*_*n*`$^{th}$ root of `*f*_*n*[*r*_1,*‚Ä¶*,*r*_n-1,*x*_*n*]==0 The represented root is `*r*_*n*`.

## Examples

### Basic Examples

Solution to a quintic:

```wolfram
Solve[x^5+2x+1==0,x]
(* Output *)
{{x->Root},{x->Root},{x->Root},{x->Root},{x->Root}}
```

Numerical values:

```wolfram

(* Output *)

```

Real solutions to an exp-log equation:

```wolfram

(* Output *)
x==Root||x==Root||x==Root
```

Real solution to a system of equations:

```wolfram
Solve[x^5-x==2&&y^5-x^2 y==3,{x,y},Reals]
(* Output *)
{{x->Root,y->Root}}
```

A solution instance to a system of transcendental equations:

```wolfram
FindInstance[Sin[x E^y]-x==1&&Gamma[x-Cos[y]]==Erf[x+y],{x,y}]
(* Output *)
{{x->Root,y->Root}}
```

### Options

#### ExactRootIsolation

[Root](https://reference.wolfram.com/language/ref/Root.html)[*f*,*k*] by default isolates the complex roots of a polynomial using validated numerical methods. Setting `ExactRootIsolation->[True](https://reference.wolfram.com/language/ref/True.html)` will make [Root](https://reference.wolfram.com/language/ref/Root.html) use symbolic methods that are usually much slower.

The setting of `ExactRootIsolation` is reflected in the third argument of a [Root](https://reference.wolfram.com/language/ref/Root.html) object:

```wolfram
a=Root[#^40-15#^17-21#^3+11&,20,ExactRootIsolation->False];
b=Root[#^40-15#^17-21#^3+11&,20,ExactRootIsolation->True];
```

```wolfram
{a,b}//InputForm
(* Output *)
{Root[11 - 21*#1^3 - 15*#1^17 + #1^40 & , 20, 0], Root[11 - 21*#1^3 - 15*#1^17 + #1^40 & , 20, 1]}
```

Root isolation is performed the first time the numerical value of the root is needed:

```wolfram
N[a,20]//Timing
(* Output *)
{0.,-0.36718277867421432680902059288843884711+1.06198756544654010857739676334979956414 ‚Öà}
```

The symbolic complex root isolation method is usually slower than the validated numeric one:

```wolfram
N[b,20]//Timing
(* Output *)
{0.09375,-0.36718277867421432680902059288849395785+1.06198756544654010857739676334972687005 ‚Öà}
```

The root isolation method may affect the ordering of nonreal roots:

```wolfram
N[{Root[#^20-15#^17-21#^3+11&,20,ExactRootIsolation->True],Root[#^20-15#^17-21#^3+11&,20,ExactRootIsolation->False]},20]
(* Output *)
{0.7958978980548581111422924905494555602+0.67049962856182151813757494301811023686 ‚Öà,0.97054056319784504469812365916816153824+0.26370857625799271442246397432414215049 ‚Öà}
```
