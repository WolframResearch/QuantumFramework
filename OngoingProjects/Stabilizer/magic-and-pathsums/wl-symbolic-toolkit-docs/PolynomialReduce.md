# PolynomialReduce | [SpanFromLeft]

> [PolynomialReduce](https://reference.wolfram.com/language/ref/PolynomialReduce.html)[*poly*,{*poly*_1,*poly*_2,…},{*x*_1,*x*_2,…}] — yields a list representing a reduction of `*poly*` in terms of the `*poly*_*i*`. The list has the form `{{*a*_1,*a*_2,…},*b*}`, where `*b*` is minimal and `*a*_1 *poly*_1+*a*_2 *poly*_2+…+*b*` is exactly `*poly*`.

## Details and Options

The polynomial `*b*` has the property that none of its terms are divisible by leading terms of any of the `*poly*_*i*`.

If the `*poly*_*i*` form a Gröbner basis with respect to the `*x*_*i*`, then this property uniquely determines the remainder obtained from [PolynomialReduce](https://reference.wolfram.com/language/ref/PolynomialReduce.html).

The result of reducing a polynomial in general depends on the ordering assigned to monomials. This ordering is affected by the ordering of the `*x*_*i*`.

The following options can be given, as for [GroebnerBasis](https://reference.wolfram.com/language/ref/GroebnerBasis.html):

| MonomialOrder | Lexicographic | the criterion used for ordering monomials |
| --- | --- | --- |
| CoefficientDomain | [Rationals](https://reference.wolfram.com/language/ref/Rationals.html) | the types of objects assumed to be coefficients |
| [Modulus](https://reference.wolfram.com/language/ref/Modulus.html) | 0 | the modulus for numerical coefficients |

[PolynomialReduction](https://reference.wolfram.com/language/ref/PolynomialReduction.html) computes just the reduced polynomial `*b*`.

## Examples

### Basic Examples

Reduce a polynomial `f` with respect to a list of polynomials `p`:

```wolfram
f=x^3+y^3;
p={x^2-y^2-1,x+2y-7};
```

```wolfram
{q,r}=PolynomialReduce[f,p,{x,y}]
(* Output *)
{{x,1+y^2},7-2 y+7 y^2-y^3}
```

`f` is a linear combination of polynomials `p` and a remainder term `r`:

```wolfram
f==q.p+r//Expand
(* Output *)
True
```

### Scope

Reduce a polynomial modulo a list of polynomials that is not a Gröbner basis:

```wolfram
f=2x^3+y^3+3y;
polys={x^2+y^2-1,x y-2};
gb=GroebnerBasis[polys,{x,y}]
(* Output *)
{4-y^2+y^4,2 x-y+y^3}
```

The remainder is not zero, even though `f` belongs to the ideal generated by `polys`:

```wolfram
PolynomialReduce[f,polys,{x,y}]
(* Output *)
{{2 x,-2 y},2 x-y+y^3}
```

When `f` belongs to the ideal generated by `polys`, the remainder modulo `gb` must be zero:

```wolfram
PolynomialReduce[f,gb,{x,y}]
(* Output *)
{{-(x)/(2)+y+(x y^2)/(2),1+x^2+(x y)/(2)-y^2-(x y^3)/(2)},0}
```

### Options

#### CoefficientDomain

By default, [PolynomialReduce](https://reference.wolfram.com/language/ref/PolynomialReduce.html) works over the field of rational functions of parameters:

```wolfram
polys={a x^2+5 y-1,2 x+x y-y^2};
poly=a^2 x-x y+y^2-3;
```

Compute the Gröbner basis of `polys` over the field of rational functions $\mathbb{Q}(a)$:

```wolfram
gb1=GroebnerBasis[polys,{x,y},CoefficientDomain->RationalFunctions]
(* Output *)
{4-16 y-19 y^2-5 y^3-a y^4,2-4 a x-9 y+(-5+2 a) y^2-a y^3}
```

Reduce `poly` modulo `gb1 over the field of rational functions $\mathbb{Q}(a)$:

```wolfram
PolynomialReduce[poly,gb1,{x,y}]
(* Output *)
{{-(1)/(4 a),-(a)/(4)+(y)/(4 a)},(2-6 a+a^2)/(2 a)-(9 (2+a^2) y)/(4 a)+((-10+4 a-5 a^2+2 a^3) y^2)/(4 a)+(1)/(4) (-2-a^2) y^3}
```

Compute the Gröbner basis and reduce `poly` over the integers:

```wolfram
gb2=GroebnerBasis[polys,{x,y},CoefficientDomain->Integers]
(* Output *)
{-4+16 y+19 y^2+5 y^3+a y^4,-2+4 a x+9 y+5 y^2-2 a y^2+a y^3,2 x+x y-y^2,-1+a x^2+5 y}
```

```wolfram
PolynomialReduce[poly,gb2,{x,y},CoefficientDomain->Integers]
(* Output *)
{{0,0,-1,0},-3+2 x+a^2 x}
```

Compute the Gröbner basis and reduce `poly` over the rationals:

```wolfram
gb3=GroebnerBasis[polys,{x,y},CoefficientDomain->Rationals]
(* Output *)
{-4+16 y+19 y^2+5 y^3+a y^4,-2+4 a x+9 y+5 y^2-2 a y^2+a y^3,2 x+x y-y^2}
```

```wolfram
PolynomialReduce[poly,gb3,{x,y},CoefficientDomain->Rationals]
(* Output *)
{{0,(a)/(4),-1},-3+(a)/(2)+2 x-(9 a y)/(4)-(5 a y^2)/(4)+(a^2 y^2)/(2)-(a^2 y^3)/(4)}
```

Compute the Gröbner basis and reduce `poly` using approximate arithmetic:

```wolfram
gb4=GroebnerBasis[polys,{x,y},CoefficientDomain->InexactNumbers[20]]
(* Output *)
{-4.+16. y+19. y^2+5. y^3+1. a y^4,-0.5+1. a x+2.25 y+1.25 y^2-0.5 a y^2+0.25 a y^3,2. x+x y-1. y^2}
```

The precision used is chosen automatically, based on the precision of the Gröbner basis:

```wolfram
PolynomialReduce[poly,gb4,{x,y},CoefficientDomain->InexactNumbers]
(* Output *)
{{0,1. a,-1.},-3.+0.5 a+2. x-2.25 a y-1.25 a y^2+0.5 a^2 y^2-0.25 a^2 y^3}
```

#### Modulus

Compute a Gröbner basis and reduce a polynomial over the integers modulo 7:

```wolfram
polys={3 x^2+y z-5 x-1,2 x+3 x y+y^2,x-3 y+x z-2 z^2};
poly=z^9-x^2 y^3-3 x y^2 z+11y z^2+x^2 z^2-5;
gb=GroebnerBasis[polys,{x,y,z},Modulus->7];
PolynomialReduce[poly,gb,Modulus->7]
(* Output *)
{{3+4 z+z^2,6,0,1},5+6 x+3 y+6 x^2 y^3+z+y z+4 x y^2 z+x^2 z^2+4 y z^2+z^3+4 z^4}
```

#### MonomialOrder

By default, [PolynomialReduce](https://reference.wolfram.com/language/ref/PolynomialReduce.html) uses the `Lexicographic` monomial order:

```wolfram
polys={3 x^2+y -5 x-1,2 x+3 x y+y^2};
poly=x y-3 x y^2 +11y +x ^3;
```

```wolfram
gb1=GroebnerBasis[polys,{x,y}]
(* Output *)
{-4-8 y+13 y^2+24 y^3+3 y^4,6+4 x+3 y-22 y^2-3 y^3}
```

```wolfram
PolynomialReduce[poly,gb1,{x,y}]
(* Output *)
{{-(169)/(144)-(53 x)/(48)+(4 y)/(3)+(5 x y)/(4)+(5 y^2)/(16)+(3 x y^2)/(16),-(13)/(24)-(3 x)/(8)+(x^2)/(4)-(7 y)/(48)-(3 x y)/(16)+(37 y^2)/(24)+(11 x y^2)/(8)+(5 y^3)/(16)+(3 x y^3)/(16)},-(13)/(9)+(85 y)/(9)+(58 y^2)/(9)+2 y^3}
```

Any `MonomialOrder` allowed by [GroebnerBasis](https://reference.wolfram.com/language/ref/GroebnerBasis.html) can be used:

```wolfram
gb2=GroebnerBasis[polys,{x,y},MonomialOrder->DegreeReverseLexicographic]
(* Output *)
{2 x+3 x y+y^2,-1-5 x+3 x^2+y,-6-4 x-3 y+22 y^2+3 y^3}
```

```wolfram
PolynomialReduce[poly,gb2,{x,y},MonomialOrder->DegreeReverseLexicographic]
(* Output *)
{{(8)/(9)-y,(5)/(9)+(x)/(3),(1)/(3)},(23)/(9)+(8 x)/(3)+(103 y)/(9)-(74 y^2)/(9)}
```

#### Tolerance

Compute approximate quotients:

```wolfram
p=x^14+3.00001*x^10-7.99998*x^7-25.00002*x^6+3.00001*x^13+9.00006*x^9-3.00001*x^5-2.00001*x^8-6.00005*x^4+16.00004*x+2.00001;
```

```wolfram
d=-2.0001465223696937 + 3.0002401628321618*x^5 + 1.*x^6;
```

With the default zero tolerance, `*d*` does not divide `*p*`:

```wolfram
PolynomialReduce[p,{d},x,CoefficientDomain->InexactNumbers]
(* Output *)
{{1. (-0.4718714366440864-8.174934686387566 x+0.058160119700057654 x^2-0.01933965088834455 x^3+3.0062258890549036 x^4-0.002071797162076516 x^5+0.0006905437730427502 x^6-0.00023016283216170663 x^7+1. x^8)},1.0561979869907394-0.3510271835774752 x+0.11632876115867542 x^2-0.03868213546816631 x^3+0.012842257450905947 x^4-1.588426261975883 x^5}
```

Increase the tolerance to obtain an approximate quotient and a zero remainder:

```wolfram
PolynomialReduce[p,{d},x,CoefficientDomain->InexactNumbers,Tolerance->0.01]
(* Output *)
{{1. (-0.998158702145961-7.99998 x+3.00001 x^4+1. x^8)},0}
```

### Applications

Test whether polynomials belong to the ideal generated by a set of polynomials:

```wolfram
f=x^6-14x^3-y^4+49;
g=x^5-y^5;
polys={x^2-y^3-5,y^2-x^3+7};
gb=GroebnerBasis[polys,{x,y}]
(* Output *)
{76-14 y^2+75 y^3-y^4+15 y^6+y^9,-1260+368 x+76 y+170 y^2-497 y^3+50 y^4+70 y^5-49 y^6+5 y^7+7 y^8}
```

The remainder is zero, hence `*f*` belongs to the ideal generated by `*polys*`:

```wolfram
PolynomialReduce[f,gb,{x,y}][[2]]
(* Output *)
0
```

The remainder is not zero, hence `*g*` does not belong to the ideal generated by `*polys*`:

```wolfram
PolynomialReduce[g,gb,{x,y}][[2]]
(* Output *)
35+5 y^2+7 y^3
```

Replace variables in a polynomial using equations relating old and new variables:

```wolfram
poly=(x+y)^2;
eqns={a==x^2+y^2,b==x  y};
gb=GroebnerBasis[eqns,{x,y}]
(* Output *)
{b^2-a y^2+y^4,b x-a y+y^3,-b+x y,-a+x^2+y^2}
```

The remainder gives a representation of `*poly*` in terms of `*a*` and `*b*`:

```wolfram
{qs,r}=PolynomialReduce[(x+y)^2,gb,{x,y}]
(* Output *)
{{(-a-2 b)/(b^2)+(y^2)/(b^2),(x)/(b)+((a+2 b) y)/(b^2)-(y^3)/(b^2),0,0},a+2 b}
```

This proves correctness of the representation:

```wolfram
poly==r/.ToRules[And@@eqns]//Expand
(* Output *)
True
```

Compute the representation of a polynomial in an algebra $\mathbb{Q}[p_{1},\ldots,p_{n}]$:

```wolfram
{p1,p2}={x^2+y^2,x  y};
```

Introduce tag variables and order them last in the monomial ordering:

```wolfram
gb=GroebnerBasis[{f1-p1,f2-p2},{x,y}]
(* Output *)
{f2^2-f1 y^2+y^4,f2 x-f1 y+y^3,-f2+x y,-f1+x^2+y^2}
```

Since the remainder is in $\mathbb{Q}[f_{1},f_{2}]$, this shows that $(x+y)^{4}\in \mathbb{Q}[p_{1},p_{2}]$:

```wolfram
Last[PolynomialReduce[(x+y)^4,gb,{x,y}]]
(* Output *)
f1^2+4 f1 f2+4 f2^2
```

Check the result:

```wolfram
%/.{f1->p1,f2->p2}//Factor
(* Output *)
(x+y)^4
```

### Properties & Relations

Reduce a polynomial with respect to a list of polynomials:

```wolfram
f=x^2-2x y^2+y^3+7;
polys={x^2-y^2-3,x y-5};
{qs,r}=PolynomialReduce[f,polys,{x,y}]
(* Output *)
{{1,-2 y},10-10 y+y^2+y^3}
```

`*f*` is equal to the linear combination of `*polys*` with coefficients `*qs*` plus the remainder `*r*`:

```wolfram
f==qs.polys+r//Expand
(* Output *)
True
```

Use [PolynomialReduction](https://reference.wolfram.com/language/ref/PolynomialReduction.html) to compute just the reduced polynomial `*r*`:

```wolfram
PolynomialReduction[f,polys,{x,y}]
(* Output *)
10-10 y+y^2+y^3
```

A polynomial belongs to the ideal generated by a Gröbner basis iff it reduces to zero:

```wolfram
{p1,p2}={x^2+y^2+z^2-1,x-2y^3-3};
```

```wolfram
{g1,g2}=GroebnerBasis[{p1,p2},{x,y,z}]
(* Output *)
{8+y^2+12 y^3+4 y^6+z^2,-3+x-2 y^3}
```

This shows that `p1 is in the ideal `{g1,g2}`:

```wolfram
PolynomialReduce[p1,{g1,g2},{x,y,z}]
(* Output *)
{{1,3+x+2 y^3},0}
```

Univariate [PolynomialReduce](https://reference.wolfram.com/language/ref/PolynomialReduce.html) is equivalent to [PolynomialQuotientRemainder](https://reference.wolfram.com/language/ref/PolynomialQuotientRemainder.html):

```wolfram
PolynomialReduce[x^5-3x+7,{2x^3-5x+11},x]
(* Output *)
{{(5)/(4)+(x^2)/(2)},-(27)/(4)+(13 x)/(4)-(11 x^2)/(2)}
```

```wolfram
PolynomialQuotientRemainder[x^5-3x+7,2x^3-5x+11,x]
(* Output *)
{(5)/(4)+(x^2)/(2),-(27)/(4)+(13 x)/(4)-(11 x^2)/(2)}
```

Reduce a noncommutative polynomial with respect to a list of noncommutative polynomials:

```wolfram
p=x**x**y**x**y+2 y**x**y**x+3 x**y;
qq={2 x**y**x+3 x**x+4 y+5,3 y**x**x+4 y**x-5 x+6};
```

```wolfram
{fs,r}=NonCommutativePolynomialReduce[p,qq,{x,y}]
(* Output *)
{{y**#1+(x**#1**y)/(2)&,-#1&},6-5 x-5 y-4 NonCommutativeMultiply+(x**y)/(2)-2 x**NonCommutativeMultiply+4 y**x-(3)/(2) NonCommutativeMultiply**y}
```

`*p*` is a linear combination of polynomials `*qq*` and the remainder polynomial `*r*`:

```wolfram
NonCommutativeExpand[p-Plus@@MapThread[Construct,{fs,qq}]-r]
(* Output *)
0
```

### Possible Issues

[PolynomialReduce](https://reference.wolfram.com/language/ref/PolynomialReduce.html) can give different results for the same inputs but with variables ordered differently:

```wolfram
polys={x^2-y^3-5,y^3-x^3+7};
newpoly=x^5+(x+y)^2;
PolynomialReduce[newpoly,polys,{x,y}]
(* Output *)
{{1+5 x+x^3+x y^3,0},5+25 x+2 x y+y^2+y^3+10 x y^3+x y^6}
```

Now change the ordering of the variables:

```wolfram
PolynomialReduce[newpoly,polys,{y,x}]
(* Output *)
{{0,0},x^2+x^5+2 x y+y^2}
```
